# Canon Entry — Odin Agent Shell v7.1

Odin Agent Shell v7.1 is a **Windows-first, local-first, protocol-bound Universal Semantic Work Kernel**, **Small-Model Performance OS**, and optional **Internal Semantic IRC/QIRC precompute engine**.

## Canon formula

```text
App-owned feature
→ Universal Work Object
→ Odin validation and precompute
→ smallest sufficient worker
→ Candidate Artifact
→ app-owned rendering and apply decision
```

## Canon laws

1. No LLM in app.
2. Every AI-like task is Universal Work.
3. Odin outputs candidates, not reality changes.
4. Apps own state, apply, sends, storage, business logic, and domain authority.
5. Models and providers are projection workers only.
6. Semantic Bus / QIRC coordinates and records traces/receipts; it does not authorize or mutate app state.
7. Local-first, remote only by explicit permission.
8. Small-model-first, 3B + 7B/8B hybrid default.
9. Larger models are escalation tiers, not default architecture.
10. Claim/evidence/reality gates are mandatory.
11. GPL-2.0-only and no-hidden-authority posture are mandatory.

## Universal Work and Candidate Artifact boundary

Odin may validate, route, precompute, compose, and return candidate-only candidate artifacts. Odin may not silently apply changes, mutate app state, perform external sends, or treat model output as truth. Caller/app-owned apply remains outside Odin.

## Claim boundary

The v0.8.7 handoff is a current canon and execution ladder lock. It is not production readiness proof, live model inference proof, model quality proof, Windows service/tray/installer proof, security certification proof, external send proof, or app-state mutation proof.

Earlier lock versions are historical/changelog context unless a v0.8.7 document explicitly binds them as traceability.
