# Bug6 / Q7 / Seed Core Synthesis v7.1

Claim boundary: this document defines architecture, contracts, shadow behavior, and Codex build intent only. It does not claim host validation, model inference proof, network operation, deployment, security audit completion, app apply, or full implementation completeness.

## Purpose

This document defines how Bug6, Q7, Y-Core posture, operational seeds, archetype roles, QIRC Gold Spine, Fairy/Y*, Shadow Runtime, and Runtime Packs combine into one coherent Odin center.

## Canonical order

```text
Universal Work received
→ Bug6 invariant precheck
→ Q7 route cleanliness scan
→ Y-Core posture selection
→ QIRC hot window
→ operational seed activation
→ archetype role synthesis
→ DFAS admissibility
→ QMath route score
→ ring radar
→ worklet/slot forge
→ model route or hold
→ candidate artifact
→ why trace
```

## Why this matters

This order moves exploration out of the model and into deterministic precompute. It improves small model utility, traceability, app safety, low-memory performance, candidate quality, and Codex build determinism.

## Combined decisions

If Bug6 blocks, no model route may proceed. If Q7 blocks, route may repair, split, or hold. If seed conflicts are unresolved, route must hold or suppress seeds. If QMath score is below threshold, route must not escalate to larger model by default. If ring radar sees no model pressure, model route must defer.

## Output contract

The synthesis outputs `odin_bug6_invariant_packet`, `odin_q7_stability_packet`, `odin_y_core_posture`, `odin_operational_seed_substrate`, `odin_seed_archetype_synthesis`, and `odin_why_trace`.

## Shadow Runtime binding

Every Shadow Runtime flow must be able to say which Bug6 invariant it preserved, which Q7 risk it reduced, which seeds were active, which archetype roles were active, and why the route was selected.

## Fairy/Y* binding

Fairy Spine provides human review. Y* Native DSL provides typed compile input. Both must carry the seed binding and Bug6/Q7 posture.

## Red lines

No Fairy/Y* node without seed binding. No Runtime Pack without Bug6/Q7 pack profile. No Candidate Artifact without Why Trace route summary. No model route without seed/archetype budget state.
