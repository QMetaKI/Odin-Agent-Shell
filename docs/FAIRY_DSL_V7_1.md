# Odin Agent Shell — v7.1 Narrative Compiler Integration

This document is part of the v0.6.0 Narrative Aorta / Y* Compiler Lock. It preserves the Odin Agent Shell v7.1 functional canon and adds a typed, bounded, auditable meta-authoring and compile-prelude layer.

Hard boundary: Fairy prose never executes. Y* Native DSL validates and stages. Shadow Runtime remains code-near blueprint. Runtime packs load only after validation. Odin Host remains the stable local boundary.

## Purpose

Fairy DSL is the human-readable narrative aorta for Odin. It exists to compress and review complex runtime meaning in a form that preserves family-first, smallest-sufficient-worker, candidate-only and app-sovereignty semantics.

Fairy DSL is not a runtime language by itself. It is a paired narrative surface that must map to Y* Native DSL nodes and from there to Shadow Runtime IR contracts.

## Dual Spine

Every Fairy story has:

1. `fairy_story`: human-readable narrative arc.
2. `ystar_unit_ref`: machine-readable typed unit.
3. `mapping`: one or more Fairy nodes mapped to Y* nodes and runtime contracts.

## Required Story Roles

- Gatekeeper: binding, policy, claim boundary and app authority.
- Weaver: context distillery, seed weaving and narrative continuity.
- Messenger: candidate response packet, never command execution.
- Lantern: trace, receipt candidate and evidence visibility.
- Small Helpers: 1B/2B/3B workers with narrow slots.
- Strong Helper: 7B/8B or quality model when route policy permits.
- House: Odin Host and pack loader boundary.

## Example Fairy Spine

Maria tells of a house called Odin. No child carries the whole forest. Each small helper receives only the basket it can safely carry. The gatekeeper checks the keys before any door opens. The weaver gathers the red threads, but she does not sew them onto the dress. The messenger returns a golden candidate, and the family decides what becomes real.

## Formal Meaning

- “No child carries the whole forest” = no small model receives raw broad context.
- “Basket it can safely carry” = slot contract + model route ladder.
- “Gatekeeper checks keys” = binding gate + policy engine + claim boundary.
- “Weaver gathers red threads” = context distillery + seed memory + worklet graph.
- “Does not sew onto the dress” = Odin does not apply.
- “Golden candidate” = Candidate Artifact + Response Packet.
- “Family decides” = app/user authority.

## Validity Rules

A Fairy story is invalid if:

- it lacks a Y* unit reference,
- it contains an unmapped narrative node,
- it implies direct app mutation,
- it implies external send by Odin,
- it implies runtime proof or security proof,
- it treats Maria/Fairy spine as public persona simulation,
- it lacks candidate-only semantics,
- it bypasses Odin Final Gate.

## Codex Rule

Codex may use Fairy DSL to understand intent and preserve architecture. Codex must implement against Y* Native DSL, Shadow Runtime IR and schemas, not against prose alone.


## no prose-only execution

Fairy DSL is a review and compression layer only. No prose-only execution is permitted. Only validated Y* Native DSL may enter the compiler prelude.
