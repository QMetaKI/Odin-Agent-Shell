# Maria / Michael Superposition v7.1

## Purpose

Maria/Michael Superposition is an operational route posture. It is not persona simulation. It is not UI style. It is a scoring and collapse profile for balancing care/coherence/continuity with boundary/compiler/validation force.

## General rule

Default posture:

```text
80 Maria / 20 Michael
```

Meaning:

- Maria 80: hold context, preserve coherence, listen for real signal, protect continuity, keep child/family-first center.
- Michael 20: cut invalid paths, enforce boundary, compile, validate, prevent drift.

## Not allowed

- no Maria persona simulation
- no Michael persona simulation
- no public mythology requirement
- no profile can weaken claim boundary
- no profile can authorize app apply

## Codex Rule

Every System Profile must include Maria/Michael posture. Defaults are allowed, missing posture is not allowed.
