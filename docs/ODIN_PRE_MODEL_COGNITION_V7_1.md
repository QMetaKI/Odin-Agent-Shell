# Odin Pre-Model Cognition v7.1

Status: Odin Agent Shell v7.1 integrated lock. This document is specification and shadow-runtime preparation only. It does not claim runtime proof, model proof, host validation, deployment, security audit completion, production readiness, app apply, or external send.

Core invariants preserved: app owns state, app owns apply, Odin returns candidate artifacts, models remain bounded projection workers, QIRC is local/internal, no hot-path runtime generation, no prose-only execution, no unvalidated runtime pack load, and all visible intelligence must remain traceable by Candidate DNA / Why Trace.


## Definition
Odin Pre-Model Cognition is the structured reasoning substrate before a model call. It is not LLM reasoning. It is deterministic and semi-symbolic preparation: context shaping, admissibility, role activation, route scoring, seed conflict resolution, output selection, and failure-mode choice.

## Cognition Units
1. Intent Center: what the user/app actually asks.
2. Boundary Center: what must not happen.
3. Evidence Center: what facts are available.
4. Resource Center: what hardware/model budget permits.
5. Output Center: what candidate shape is needed.
6. Safety Center: Bug6/Q7/Maria-Michael posture.

## Pre-Model Cognition Trace
Every significant decision is recorded as machine-readable trace:
- centerline selected
- active seeds
- active archetype roles
- rejected routes
- model work avoided
- required model slot if unavoidable
- expected user-visible effect
- why this is candidate-only

## Validation
Pre-model cognition is valid only if it produces a bounded candidate path and preserves app-owned authority. If uncertainty is too high, it must choose ASK_CONTEXT, SPLIT_WORK, HOLD, or BLOCK instead of forcing model dispatch.

## DFAS Stability Core Binding
The DFAS Stability Core is the admissibility partner for Odin Pre-Model Cognition. It determines whether the work continues as GO, HOLD, ASK_CONTEXT, SPLIT_WORK, or BLOCK before any model receives a packet.
