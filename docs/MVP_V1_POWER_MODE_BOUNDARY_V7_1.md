# MVP / V1 / Power Mode Boundary v7.1

## Purpose

Separate build phases so Codex does not attempt to implement the full architecture before the minimal spine is stable.

## MVP

MVP implements:

- flat public repo canon;
- validation CLI;
- schemas and registries;
- mock provider;
- Universal Work validation;
- Candidate Artifact return;
- minimal QIRC event batch;
- minimal Odin Hub status panel;
- app pairing scaffold;
- support bundle scaffold;
- GPL-2.0-only policy.

MVP excludes:

- real remote workers;
- heavy model modes;
- full Control Center richness;
- Pattern Mine live import;
- runtime pack compiler execution;
- Windows installer beyond portable profile;
- agent tool execution;
- app apply.

## V1

V1 adds:

- Ollama and llama.cpp adapters;
- standard local 3B + 7B/8B hybrid route;
- QIRC Gold Spine basic channels;
- Seed Pack compiler;
- Work Atom runtime;
- Candidate Canvas;
- Why Trace;
- safe mode;
- portable installer packaging.

## Power Mode

Power Mode adds:

- Pattern Mine intake;
- runtime pack AOT/cached capability slices;
- advanced QIRC channels;
- Model Dojo and Scoreboard;
- Shadow Narrative red-team compiler;
- large local/heavy model routes;
- deep Hub panels;
- developer diagnostics.

## Boundary rule

Power Mode may never weaken MVP safety. Every higher mode inherits candidate-only, app-owned apply, GPL-2.0-only, local-first defaults and claim boundary.
