# Odin QIRC Gold Spine v7.1

## Purpose

Odin QIRC Gold Spine is the local-only semantic event backbone that makes Odin Core, QLI Master Interface, DFAS Stability Core, Seed / Archetype Economy, QMath Center Solver, Ring Radar / Resonance, Shadow Runtime, Runtime Packs, Model Routing, Critics, Candidate Assembly and Why Trace work as one bounded system.

It is not public IRC. It is not LAN mesh. It is not phone mesh. It is not app-state ownership. It is not an agent chat room. It is the reason the model has less to guess.

## Canonical Formula

```text
Universal Work
→ QIRC ingress event
→ hot-window context
→ seed prewarm
→ archetype role activation
→ admissibility gate
→ QMath center / route score
→ ring radar / resonance activation
→ worklet / slot dispatch
→ model route or hold/split/ask/block
→ critic / tournament flow
→ candidate assembly
→ why trace / receipt candidate
```

## Authority Boundary

QIRC may coordinate internal Odin work. QIRC may not grant authority, mutate app state, publish app mutation events, bypass Odin Final Gate, dispatch external sends, or claim runtime proof.

## Gold Spine Layers

- QIRC-L0 Transport: in-process or local-only semantic event transport.
- QIRC-L1 Typed Event Envelope: every event is typed, traced and privacy-classed.
- QIRC-L2 Context Backbone: raw digest to hot-window snapshot.
- QIRC-L3 Seed / Archetype Prewarm: activate, decay, conflict-resolve and cap seeds.
- QIRC-L4 Centerline / Admissibility: DFAS hold/go/split/ask/block.
- QIRC-L5 Ring Radar / Resonance: activate only the relevant rings.
- QIRC-L6 Worklet / Slot Dispatch: build worklets before model calls.
- QIRC-L7 Model Route Dispatch: route smallest sufficient worker.
- QIRC-L8 Critic / Candidate Flow: chain critics and tournaments.
- QIRC-L9 Why Trace / Receipt / Replay: explain decisions and enable replay.
- QIRC-L10 App-QIRC Digest Bridge: digest-only bridge for app-owned event systems.

## Why This Improves Odin

The bus gives Odin a cheap, typed, replayable and inspectable precompute surface. It reduces context size, model calls, retry churn and authority drift while improving traceability, small-model precision and Codex buildability.

## Red Lines

- No QIRC network expansion in core.
- No public IRC.
- No app-state mirror.
- No raw secret payloads.
- No model dispatch before admissibility.
- No unlimited seed activation.
- No ring fanout without pressure threshold.
- No QIRC event without trace_id.
- No why trace without route reason.
- No QIRC channel that can mutate app state.
- No app-QIRC bridge beyond digest mode by default.

## Implementation Rule

QIRC Gold Spine is implemented as contracts first, schemas second, registries third, shadow modules fourth, tests fifth, real runtime later. The implementation must remain boring, local, typed, replayable and bounded.
