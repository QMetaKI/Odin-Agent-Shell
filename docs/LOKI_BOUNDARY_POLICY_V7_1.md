# Loki Boundary Policy v7.1

## Purpose
Defines what the Loki mediation layer may and may not do.

This document is part of Odin Agent Shell v7.1 / v0.7.0 Shadow Narrative Loki Anti-Pattern Lock. It is specification, shadow-runtime, validation and Codex-build guidance only. It does not claim runtime execution, model inference, host validation, network operation, deployment, security certification, app apply, external send, or full implementation completeness.

Core invariant: Odin remains candidate-only. App state, apply, external sends and domain reality remain app-owned. Loki/Shadow Narrative may reveal risk and propose gates; it may not decide authority or bypass Odin Final Gate.


## Allowed
- reveal ambiguity
- score drift risk
- propose anti-pattern gates
- recommend narrow/split/hold/block
- generate candidate negative tests
- annotate Why Trace

## Forbidden
- deciding final route
- granting capability
- executing model calls
- mutating app state
- issuing receipts
- accepting claims
- bypassing Odin Core
- changing runtime pack load policy

## Policy Precedence
Bug6 and Children-First win. Q7 Stability wins. Odin Core Centerline wins. App Apply Boundary wins. Claim Boundary wins. Loki may only add caution, never subtract boundary.

## Archetypic Meaning
Loki represents the necessary trickster mirror: not evil, not authority, but the one who reveals hidden cost. This is useful only because Odin Core remains final authority.
