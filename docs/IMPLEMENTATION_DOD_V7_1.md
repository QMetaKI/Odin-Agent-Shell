# Implementation Definition of Done v7.1 — Deep Subsystem Spec

**Spec Status:** v7.1 / v0.3.2 DEEP_SUBSYSTEM_SPEC_LOCK.  
**Authority:** build contract for Codex and later implementation work.  
**Claim Boundary:** specification only; no runtime, host, model, security or deployment proof is implied.


## Implementation Definition of Done Purpose

This document defines when Codex may consider a subsystem implemented for prep-stage acceptance. It is not a final release certification; it is a build-stage readiness contract.


## DoD — Universal Work Kernel

A PR implementing `Universal Work Kernel` is complete only when:

```text
- public interface is documented
- schema/registry references are updated
- validator rejects malformed input
- valid example exists where applicable
- negative example exists where boundary-sensitive
- tests cover success and failure path
- trace or reason code is emitted for failures
- no app-state mutation is introduced
- candidate-only posture is preserved
- SYSTEM_MAP and FILE_MANIFEST are updated
```

Forbidden shortcuts:

```text
- accepting arbitrary dicts without validation
- direct model prompt bypassing ModelWorkPacket
- returning raw model text as final app action
- adding remote behavior without policy gate
- adding semantic bus network exposure by default
```
