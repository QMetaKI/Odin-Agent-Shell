# QFoundation System Intake for Odin v7.1

## Purpose

This document binds QFoundation-derived patterns into Odin without importing QFoundation naming into the public surface. The result is an Odin System Palette that improves intent routing, authority posture, ring path selection, Maria/Michael biasing, seed/archetype precompute, synthesis collapse and claim-bound output control.

## Source Reading Summary

QFoundation defines itself as a private ChatGPT-/agent-facing Q/standard-system foundry and compiler-like preordering repository. Its route is explicitly: Intent → Authority Posture → Ring Path → Maria/Michael Bias → Ambivalence Matrix → Composer → System Depth Profiles → Archetype/Seed Mechanics → Synthesis Collapse → Precompute Packet → FNC/CSIR/QCode Handoff or Normal Output → Claim Boundary.

QFoundation's SYSTEM_MAP shows the system breadth that matters to Odin: registries for Q systems, standard systems, bridges, sources, agents, domains, claims, task packs, composition, ambivalence, fractal navigation, output translation, agent use, quality, playbooks, recipes and route maps. This maps cleanly to Odin as a local LLM-work foundry: instead of app/product systems, Odin composes LLM-work routes.

The intent-to-Q-system/standard matrix is particularly valuable because it binds every intent family to authority posture, ring path, Maria/Michael bias, middle-out center, seeds, synthesis collapse rule, output mode and claim boundary. For Odin, this becomes the basis of a neutral System Palette where each Universal Work gets a route profile.
